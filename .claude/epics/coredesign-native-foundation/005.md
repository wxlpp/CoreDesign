# Task: 组件机械改名扫尾并删除弃用别名

## Description

把全部组件与 modifier 的调用点从旧 token 名迁到新名，然后删掉 Task 003 留下的弃用别名。

这一段是**机械的、由编译器驱动的**：Task 003 的 `@available(*, deprecated, renamed:)` 会为每个旧名调用点产生一条 warning，逐条消除即可。与 Task 006 的判断密集型工作拆开，是因为二者的验证方式完全不同——本任务的完成判据是「warning 清零、别名删净」，006 的判据是「每个档位选择都经过确认」。

## Acceptance Criteria

### 改名迁移

- [ ] 全部保留组件的旧字体 token 调用迁到新名（基线 63 处：`.bodyMedium` 16、`.bodySmall` 18、`.caption` 11、`.bodyLarge` 6、`.captionMono` 4、`.titleSmall` 4、`.titleLarge` / `.titleMedium` 各 2）
- [ ] `Sidebar.swift:157,411` 的 `CoreRadius.mediumPlus` 迁到新档位
- [ ] `primerVerticalPadding(for:)` 的调用点（若有）迁走
- [ ] 覆盖范围：`Components/`（Avatar / AvatarGroup / Badge / Banner / Tag / ListRow / Form / SearchField / SegmentedControl / Sidebar / UnderlinedTabBar / Toast / ProgressBar / ProgressIndicator / CheckBox / StateLabel / BottomInputBar / CoreMenuButton / AsyncButton / 4 个 Button style / StatusLevel）、`Layout/FlowLayout.swift`、`Shape/StarShape.swift`、`Modifier/` 全部 8 个文件、`Utils/ColorExtension.swift`

### 别名清理

- [ ] `CoreTypography` 的 12 个 `@available(deprecated)` 旧名别名删除
- [ ] `CoreRadius.smallPlus` / `mediumPlus` 删除
- [ ] `CoreControlMetrics.primerVerticalPadding(for:)` 删除

### 验证

- [ ] `swift build` 无 deprecation warning
- [ ] `swift build` / `swift test` 通过，CI 全部命令通过

## Technical Details

本任务**不做视觉判断**——不评估圆角是否该翻倍、高度是否该从 32 变 44，那些是 Task 006 的事。这里只保证「旧名不再被引用、别名已删净、树是绿的」。

改名迁移与 Task 006 的逐点重审必须分开，否则执行者会在同一遍里既做机械替换又做设计判断，结果通常是机械替换做完了而设计判断被跳过。
