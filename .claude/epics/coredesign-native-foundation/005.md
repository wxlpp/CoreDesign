# Task: 保留组件逐调用点迁移到新 token

## Description

把全部保留组件迁到 Task 003 / 004 定案的新 token 上。这是本 epic 最大的一块，也是「同名换值」风险的兑现点。

产出**不只是「改完能编译」**——同名换值对编译器完全透明，必须产出一份「旧档位 → 新档位」的逐调用点迁移映射表，逐点确认新档位是有意选择而非自动继承。

## Acceptance Criteria

### 迁移范围

- [ ] 保留组件全部迁移：Avatar / AvatarGroup / Badge / Banner / Tag / ListRow / Form / SearchField / SegmentedControl / Sidebar / UnderlinedTabBar / Toast / ProgressBar / ProgressIndicator / CheckBox / StateLabel / BottomInputBar / CoreMenuButton / AsyncButton / 四个 Button style / FlowLayout / StarShape
- [ ] `Modifier/` 下全部 modifier 迁移（BorderModifier / ButtonBackgroundModifier / ButtonChromeModifier / CoreFontModifier / FloatingGlassModifier / FocusRingModifier / SurfaceModifier / TelegramGlassButtonModifier）
- [ ] `Utils/ColorExtension.swift` 按新语义色调整

### 同名换值逐点确认（本任务的核心产出）

- [ ] 产出迁移映射表，记录每个调用点的旧档位 → 新档位 → 确认理由
- [ ] 重点覆盖圆角：新 `small = 6` 恰等于旧 `medium`，所有按旧语义选 `small = 3` 的调用点（Badge、Tag、`ProgressBar.swift:45,47`、`SearchField.swift:86` 等）圆角会静默翻倍——逐点确认这是想要的效果
- [ ] 重点覆盖控件高度：`regular` 32 → 44 对布局的连锁影响
- [ ] 重点覆盖语义色指向变更（清单由 Task 004 提供）

### 圆角出口收敛

- [ ] `grep -rn "RoundedRectangle(" Sources` 中裸调用为 0——全部经 Task 003 的 shape helper 或 `ConcentricRectangle`
- [ ] 嵌套于已知容器的元素改用 `ConcentricRectangle`

### 触控目标

- [ ] 全部交互组件在 `.regular` 控件尺寸下实测可点击高度 ≥ 44pt
- [ ] 补充测试覆盖该断言（注意：须跑在 CI 的 xcodebuild iOS Simulator 腿上）

### Dynamic Type

- [ ] `DynamicTypeLayoutTests` 覆盖新 token
- [ ] **翻转 `captionSmallDoesNotScale`（`DynamicTypeLayoutTests.swift:73-74`）**——它编码的是旧的「9pt 不缩放」契约，与新 NFR 相反；该 suite 其余断言是单调性检查，对重铸稳健
- [ ] 最大辅助功能字号下布局不裁切、不重叠

### 通用

- [ ] `swift build` / `swift test` 通过
- [ ] CI 全部命令通过，含 xcodebuild iOS Simulator 腿
