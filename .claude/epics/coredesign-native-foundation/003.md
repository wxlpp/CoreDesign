# Task: 重铸字体 / 圆角 / 尺寸 / 阴影 token

## Description

把 `Tokens/` 下的度量类 token 从 Primer 标度换成 Apple HIG 标度。本任务只改 token **定义**，不动组件调用点——调用点迁移是 Task 005 / 006 / 007，拆开是为了让「新档位定案」与「逐点确认新档位用对了」成为两次独立的判断。

**必须走弃用别名过渡，不得硬切**。旧字体 token 名在 `Tokens/` 之外有 63 处引用（`.bodyMedium` 16、`.bodySmall` 18、`.caption` 11、`.bodyLarge` 6、`.captionMono` 4、`.titleSmall` 4、`.titleLarge`/`.titleMedium` 各 2），删 `mediumPlus` 直接打断 `Sidebar.swift:157,411`。硬改名会让 `swift build` 从本任务一直红到迁移完成（约 36 小时），期间 CI 与每个 checkpoint 评审全部失去把关能力，且 Task 004 的「build 通过」无从演示。

因此：新名与旧名并存，旧名标 `@available(*, deprecated, renamed:)` 指向新名。全程 build 绿，每个任务独立可验证。别名由 Task 005 在迁移完成后删除。

## Technical Details

**`.continuous` 必须经 helper 强制**：只改 `CoreRadius` 数值拿不到 Apple 观感，因为角样式要在每个 `RoundedRectangle` 构造点显式指定，漏一处就是一个风格不一致的元素。因此验收判据是「grep 裸 `RoundedRectangle` 为 0」（Task 006 达成），而本任务负责提供那个唯一出口。

**同名换值不能靠别名兜住**：新 `small = 6` 恰等于旧 `medium`，`regular` 高度 32 → 44。这类改动**不产生 warning 也不产生编译错误**，调用点静默继承新值——与改名类不同，它无法由 deprecation 机制发现。本任务须在交付说明中列出全部换值项，交 Task 006 逐点重审。
