# Task: 重铸语义色层与 accent 衍生族

## Description

把第 3 层语义色 token 在存在 Apple 对应物时改为指向系统色，使库跟随用户的系统外观设置。这是「与系统融为一体」最直接的兑现。

本任务是本 epic 里判断密度最高的一块：它不只是改指向，还要处理 `accent` 衍生族在动态强调色下无从推导的问题、macOS 缺少 grouped background 系列的降级、以及 `ButtonRoleStyleRole` 调色板的重写。

## Technical Details

**不承诺跟随每视图 `.tint(_:)`**：`.tint` 走的是独立于 `Color.accentColor` 的 `ShapeStyle` 通道；要让静态色 token 跟随它，需把组件配色整体改成 `ShapeStyle` 通路，属 API 形态变更，不在本 epic。需要按视图 tint 的场景走 SwiftUI 原生通路——这条逃生口的兑现落在 Phase 2 的 `.core` style 套件上（那里有硬约束：不得写死静态 accent）。

**macOS 现状**：`SystemBackgroundColors.swift:47-53` 与 `58-64` 两个 grouped background 在 macOS 侧都落 `.controlBackgroundColor`。照搬 iOS 映射会让卡片与画布同色、raised 层完全隐形——这是功能性退化，不属于「macOS 观感不打磨」的可接受范围。
