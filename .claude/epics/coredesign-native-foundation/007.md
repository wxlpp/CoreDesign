# Task: 触控目标与 Dynamic Type 验证

## Description

把 token 重铸带来的两条可访问性承诺兑现成测试：44pt 触控下限，以及文字随 Dynamic Type 缩放。

这两条是 PRD 里少数**可以被断言证明**的原生化指标——前面几个任务的判据多是「grep 为 0」这类结构性检查，能证明代码长什么样，不能证明它在设备上的行为对。

## Acceptance Criteria

### 触控目标

- [ ] 全部交互组件在 `.regular` 控件尺寸下实测可点击高度 ≥ 44pt
- [ ] 补充测试覆盖该断言
- [ ] **测试须跑在 CI 的 xcodebuild iOS Simulator 腿上**——macOS 的 `swift test` 里 `#if os(iOS)` 的 suite 是空的，只跑 `swift test` 等于没跑

### Dynamic Type

- [ ] `DynamicTypeLayoutTests` 覆盖全部新 token
- [ ] （`captionSmallDoesNotScale` 的翻转已在 Task 003 完成——它必须与 token 定义同任务，否则 CI iOS 腿从 003 起持续红）本任务负责该 suite **其余断言对新 token 的扩充覆盖**；原有的单调性检查对重铸稳健，无需改写
- [ ] 最大辅助功能字号下布局不裁切、不重叠

### 验证

- [ ] `swift build` / `swift test` 通过
- [ ] **CI 全部命令通过，含 xcodebuild iOS Simulator 腿**

## Technical Details

`DynamicTypeLayoutTests.swift:8` 整个 suite 是 `#if os(iOS)`，只在 CI 中**唯一那条 `xcodebuild` iOS Simulator 命令**下执行（Task 002 删掉两条 Blossom 命令后其序号与行号都会变，故此处不用行号指代）。本任务新增的触控断言若也写在 iOS-only suite 里，同样只活在那条腿上——所以验收必须以「CI 全部命令通过」为准，不能以本地 `swift test` 绿了为准。
