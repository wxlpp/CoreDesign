# Task: CoreControlMetrics token

## Description

新增 `Sources/CoreDesign/Tokens/CoreControlMetrics.swift`，按 SwiftUI `ControlSize`（mini / small / regular / large / extraLarge）暴露查询表 helper：高度 / 横向 padding / 纵向 padding / 推荐字号（来自 CoreTypography）/ icon 尺寸。后续 components epic 中按钮、SegmentedControl、SearchField 等控件统一从此表读取尺寸。

## Acceptance Criteria

- [ ] 新建 `CoreControlMetrics.swift`，caseless `public enum CoreControlMetrics`
- [ ] 提供 5 个 helper：`height(for:)` / `horizontalPadding(for:)` / `verticalPadding(for:)` / `font(for:)` / `iconSize(for:)`，每个接收 `ControlSize` 返回对应 `CGFloat` 或 `Font`
- [ ] 各 helper 内部使用 `switch` 分发，处理全部 5 个 ControlSize case + `@unknown default`
- [ ] padding 取值引用 `CoreSpacing.*`；font 取值引用 `CoreTypography.*Font`，**不内联魔法数字**（这是本 token 存在的意义）
- [ ] 文件顶部含注释 `// Source of truth: docs/PRIMER_VERSION.md`
- [ ] 每个 helper 上方 doc-comment 简要说明取值依据（譬如"按 Primer control size 表，regular = 32pt 高度"）

## Technical Details

- 现有 `SolidButtonStyle.swift:42` 的 EdgeInsets 表是真实参考依据：`mini=2/12, small=2/12, regular=6/12, large=8/16, extraLarge=10/20`。验证这些值与 Primer control sizes 对齐情况，必要时调整以与 Primer 一致（具体 Primer 数值在 Task 1 锁定的版本中查证）
- `ControlSize.mini` 与 `ControlSize.small` 在现有按钮中取值相同；本任务可保持，也可按 Primer 拉开差距，由实施者决定
- 不需要支持 `extraLarge` 之外的扩展尺寸
