# Task: SurfaceModifier

## Description

新增 `Sources/CoreDesign/Modifier/SurfaceModifier.swift`，实现 `View.surface(_:)` modifier 与配套 `SurfaceKind` 枚举，让调用方一次性应用"背景 + 1px 边框 + 圆角"组合，避免 Sidebar / Panel / Card 这类容器场景每次手写 `RoundedRectangle().fill().overlay(stroke())` 三件套。

## Acceptance Criteria

- [ ] 新建 `SurfaceModifier.swift`
- [ ] 暴露 `public enum SurfaceKind { case canvas, canvasSubtle, panel, sidebar, card }`
- [ ] 暴露 `public extension View { func surface(_ kind: SurfaceKind) -> some View }`
- [ ] 每个 SurfaceKind 内部映射到一组 `(background: Color, border: Color, cornerRadius: CGFloat)`，全部从 token 派生：
  - background 引用 Task 5 新增的 surface 色（`.surfaceCanvas` / `.surfaceCanvasSubtle` / `.surfacePanel` / `.surfaceSidebar` / `.surfaceCard`）
  - border 引用 Task 5 新增的 border 色（`.borderMuted` / `.borderDefault` 等，按 Primer 视觉决定）
  - cornerRadius 引用 `CoreRadius.*`（建议 panel/card = `.medium`，sidebar = `.none`，由实施者按 Primer 决定）
- [ ] 提供 `#Preview` 一次性展示全部 5 个 SurfaceKind 的视觉差异（垂直堆叠 + 标签）
- [ ] light + dark colorScheme 双预览

## Technical Details

- 内部实现可用 `Rectangle().fill(bg).overlay(RoundedRectangle().stroke(border)).clipShape(RoundedRectangle())` 三件套；或复用现有 `.bordered(...)` modifier
- 注意 `.clipShape` 对 shadow 的影响——本 modifier **不应**叠加 shadow（shadow 由调用方按需追加 `.coreShadow(_:)`）
- `SurfaceKind` 命名维度统一为"具体容器 / 具体容器变体"（按 PRD G-2 修正后的命名），不引入裸修饰词
