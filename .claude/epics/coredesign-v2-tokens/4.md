# Task: CoreElevation token + dark-adaptive shadow colorset

## Description

新增 `Sources/CoreDesign/Tokens/CoreElevation.swift`，提供 4 档阴影 token + `View.coreShadow(_:)` 便利 modifier；在 `Resources.xcassets/shadow/` 顶级目录下新建 light/dark 双取值的 shadow colorset，确保暗色模式下阴影仍可辨识（Primer 与 Apple HIG 在 dark 模式下用更高不透明度替代投影，避免 elevation 视觉消失）。

## Acceptance Criteria

- [ ] 新建 `Resources.xcassets/shadow/` 顶级目录（与现有色相目录平级，按 epic Architecture Decision #7 命名约定）
- [ ] 该目录下新增 ≥4 个 colorset，对应四档 elevation：`shadow-none.colorset` / `shadow-small.colorset` / `shadow-medium.colorset` / `shadow-large.colorset`
- [ ] 每个 shadow colorset 配置 light + dark 两套值（dark 模式取值的不透明度需 ≥ light 的 2 倍，否则视觉消失；具体倍数按视觉验证调整）
- [ ] 新建 `CoreElevation.swift`，caseless `public enum CoreElevation`，含 `Level` 子枚举（`.none` / `.small` / `.medium` / `.large`）与每档对应的 `(shadowColor: Color, radius: CGFloat, x: CGFloat, y: CGFloat)` 元组
- [ ] 暴露 `public extension View { func coreShadow(_ level: CoreElevation.Level) -> some View }` 便利 modifier，内部调用 `.shadow(color:radius:x:y:)` 应用对应取值
- [ ] 文件顶部含注释 `// Source of truth: docs/PRIMER_VERSION.md`
- [ ] 提供 `#Preview` 在 light + dark 两种 colorScheme 下展示四档 elevation 视觉差异（用 `.preferredColorScheme(.dark)` 单独输出 dark 预览）
- [ ] **人工验证**：dark mode 下 `.medium` 与 `.none` 视觉差异可辨识

## Technical Details

- shadowColor 通过 `Color("shadow-medium", bundle: .module)` 加载 colorset 中的动态色
- Primer shadow tokens 见 `https://github.com/primer/primitives/tree/<locked-version>/src/tokens/functional/shadow`
- 单层 shadow 通常已能满足 Primer 视觉；如需多层 shadow（譬如 Primer 的 shadow-large 是双层组合），考虑在 `coreShadow` modifier 内连续调用两次 `.shadow(...)`
- 颜色模式切换由 SwiftUI 自动驱动（colorset 的 light/dark 配置）
