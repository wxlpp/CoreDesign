# Task: BorderModifier canary 迁移

## Description

将 `Sources/CoreDesign/Modifier/BorderModifier.swift` 内的 3 处魔法数字字面量替换为 token 引用，作为"token 在现有代码中能跑"的最小端到端证明。本任务是本 epic 内**唯一允许动到现有组件的例外**（其他组件改造留给 components epic）。

## Acceptance Criteria

- [ ] `BorderModifier.swift` body 内 `RoundedRectangle(cornerRadius: 0)` 改为 `RoundedRectangle(cornerRadius: CoreRadius.none)`
- [ ] `View.bordered(style:width:)` overload 的默认 `width: CGFloat = 1` 改为 `width: CGFloat = CoreBorderWidth.thin`
- [ ] `View.bordered(color:width:)` overload 的默认 `width: CGFloat = 1` 改为 `width: CGFloat = CoreBorderWidth.thin`
- [ ] 不改 API，不改行为；值语义完全等价（`CoreRadius.none == 0`，`CoreBorderWidth.thin == 1`）
- [ ] grep 验证：`grep -E 'cornerRadius:\s*0|width:\s*CGFloat\s*=\s*1' Sources/CoreDesign/Modifier/BorderModifier.swift` 无匹配

## Technical Details

3 处字面量分别在：

1. `BorderModifier.body` 内的 `RoundedRectangle(cornerRadius: 0)` 字面量 0
2. `func bordered(style:width:)` 的 `width: CGFloat = 1` 默认参数
3. `func bordered(color:width:)` 的 `width: CGFloat = 1` 默认参数

引用 token 后无需 import 额外模块（`CoreRadius` / `CoreBorderWidth` 在同一 CoreDesign target 内可见）。
