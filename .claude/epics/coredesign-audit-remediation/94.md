# Task: 公开 API 修复与改名

## Description

承载 6 个审计项：A2a、A2b、A2c、A3a、A3b、B9e（对应 FR-2）。

两条主线：

1. **补 `public`**——一批公开 API 因漏写 `public` 而下游不可用，已用下游消费包实测出编译错误（`cannot find in scope` / `initializer is inaccessible`），其中包括文档明确指引业务侧使用的 `CheckBoxToggleStyle`。
2. **改名避开 SwiftUI 同名冲突**——`BorderlessButtonStyle` 与 `MenuButton` 都与 SwiftUI 内建类型同名。这比漏写 `public` 更隐蔽：下游写这两个名字**能编译通过**，但静默解析到 SwiftUI 版本。

## Technical Details

### 两处改名的必要性（静默解析，非编译错误）

- **`BorderlessButtonStyle`**：与 SwiftUI 自带类型同名。下游写 `BorderlessButtonStyle()` **能编译通过**，但拿到的是 SwiftUI 的类型，不是 CoreDesign 的。改名为 `CoreBorderlessButtonStyle` 后不存在「解析到 SwiftUI 版本」的可能。
- **`MenuButton`**：与 macOS 上 deprecated 的 SwiftUI `MenuButton` 冲突，实测报错信息具误导性。改名为 `CoreMenuButton`。

改名须连带更新：类型声明、`ButtonStyle where Self == ...` 上的静态扩展、全部内部消费点、`#Preview`、docs 中的示例、**以及 `App/` 预览宿主**（仓库内唯一需同步适配的消费者）。

### `CheckBox` 的处置（B9e）

`CheckBox.swift:53-59` 是硬编码 `Toggle("哈哈哈哈哈")` 的演示视图，用 `@State` 而非 `@Binding`，唯一使用者是同文件 Preview。处置：**内联进 `#Preview`**，公开面只保留 `CheckBoxToggleStyle`。

注意 `CheckBox.swift` 同时是 #2 的触及点（A1 的 `:31` 改 `Color.contentPrimary`、`:21-23` 注释修正）——这正是本任务 `depends_on: [002]` 的原因，两者串行。

### 范围边界：`View.getSize` 不在补 public 之列

`Utils/View+SizeReader.swift:44,48` 的 `getSize` 同样被实测确认下游不可达，但它的处置是**删除**而非补 `public`——它随 #6 的 FR-5 死代码清理一并移除（全库唯一消费点 `BottomInputBar.swift:138` 也在 #6 删除范围内）。**本任务不要给它补 public**，否则 #6 会白删一个刚公开的 API。

### 触及文件

`CheckBox.swift`、`BorderlessButtonStyle.swift`（改名后文件名亦须改）、`ButtonRoleStyleRole.swift`、`MenuButton.swift`（同上）、`App/` 预览宿主、相关 docs。
