## Technical Details

### 删除的验证方式

每个整文件删除后须 grep 确认零残留引用，不能只靠编译通过——公开符号被删除时下游不在本仓，编译器不会报错。删除清单：

| 删除对象 | 唯一消费点 / 依据 |
|---|---|
| `EmptyState.swift` + `EmptyStateDeprecationTests.swift` | 全文件已 deprecated，测试是自证测试 |
| `View+SizeReader.swift` | `getSize` 唯一消费点 `BottomInputBar.swift:138`（本任务同时删除） |
| `KeyboardHandling.swift` + `KeyboardHandlingTests.swift` | 生产调用点为零；`KeyboardHeightPublisherFactory` 仅测试消费 |

### 与 007 的交接

`KeyboardHandlingTests.swift` 随本任务删除，**不在 007 的「保留测试」名单之列**。007 的逐文件处置清单须以此为前提。

### 依赖说明

- **002**：色彩层重组会改写 `CommentCard` / `RefPill` 等文件所依赖的语义 token，且 `CheckBox.swift` 是 002 与 003 的共享文件；本任务的 B9f 同样改 `CheckBox.swift`
- **003**：`CheckBox` 演示视图内联进 `#Preview`（B9e）由 003 完成，本任务的 B9f 须在其后

### 与 005 并行的硬约束

005 ∩ 006 = ∅ 是矩阵验算结论，**执行时不得破坏**：本任务不得触碰 005 的自有文件（四个 ButtonStyle、`ButtonRoleStyleRole.swift`、`Sidebar.swift`、`MenuButton.swift`、`TelegramGlassButtonModifier.swift`）。B7a 的消费点选择是这条约束下最容易失手的一处，已在 Acceptance Criteria 中钉死。

### 代码风格

遵循仓库既有约定：显式 `self.`、中英双语注释与 `// MARK: -`、`#Preview` 与组件同文件。保留有设计说明价值的长注释（如 `BottomInputBar.autoFocus` 解释为何必须在 bar 自身 `onAppear` 中执行）——拆分 `body` 时不要连带丢弃。
