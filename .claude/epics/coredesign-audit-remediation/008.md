# Task: 可访问性

## Description

补齐三处 VoiceOver 可访问性缺口，承载审计项 **D1a、D1b、D1c** 共 3 项（对应 FR-7 的 a11y 部分、US-5）。

三处都是同一类问题的不同形态：**图标承载的信息没有暴露给辅助技术**——icon-only 按钮无标签、选中态未播报、装饰性图标未隐藏。三处都在同仓已有正确范例，本任务是把既有做法补齐到遗漏处，不引入新模式。

本任务属于 epic 的**并行窗口 2**：owner 矩阵逐对验算确认 **008 ∩ 010 = ∅**，可与 010（公开 API 形态统一）真正并发。

## Technical Details

### 三处的处理原则不同，不要一律照抄

| 图标类型 | 处理 | 本任务实例 |
|---|---|---|
| icon-only 可交互控件 | `accessibilityLabel` | `BottomInputBar` 三个按钮、`Form.DangerIcon` |
| 纯装饰、信息已由邻近文本承载 | `accessibilityHidden(true)` | `Form.LabelIcon`、`Form.ChevronRightIcon` |
| 状态需播报 | `accessibilityAddTraits` | `UnderlinedTabItem` |

`DangerIcon` 是最容易误判的一处——它与另两个 `Form` 图标同文件、形态相似，但它**承载语义**（危险状态本身就是信息），必须给 label 而非隐藏。

### 字符串处理

本任务补的 accessibility 标签属新增 UI 字符串。**本轮直接写英文字面量**，String Catalog 迁移由 009（本地化）负责——009 排在本任务之后，会把这批字符串一并纳入。不要在本任务里自行引入 `.xcstrings`（那会触碰 `Package.swift`，属 009 与 001 的范围）。

### 依赖说明

- **002**：`Form.swift:101` 使用 legacy `Color.dangerForeground`，由 002 迁移到新 `StatusColors` 体系；本任务改同一文件，须在其后
- **004**：Dynamic Type 改造波及 `Form.swift` 与 `BottomInputBar.swift`（`:302,376` 的 D3 系统字号迁移），须在其后
- **006**：`BottomInputBarModifier.body` 的拆分（B8h）会移动本任务要改的三个按钮，须在其后——否则两边改同一 hunk

### 代码风格

遵循仓库既有约定：显式 `self.`、中英双语注释与 `// MARK: -`。
