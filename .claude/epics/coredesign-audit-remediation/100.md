# Task: 本地化 String Catalog

## Description

本任务承载 1 个审计项：**D2**（硬编码中文 UI 字符串，与别处英文不一致；全库无 String Catalog）。它是全 epic 承载项最少、触及文件最少的任务（1 项 / 3 个组件文件 + `Package.swift`），但**含一处跨任务触碰**，须谨慎处理。

库内 UI 字符串目前是分裂的：一部分组件已经是英文（`ProgressBar` 的 "Progress"、`ProgressIndicator` 的 "Loading"、`Tag` 的 "Remove tag"、`AvatarGroup` 的 "more avatars"），另外四处是硬编码中文：

| 位置 | 当前字符串 | 性质 |
|---|---|---|
| `Toast.swift:441` | `"点击关闭"` | 可访问性 / 提示文案 |
| `CoreMenuButton`（原 `MenuButton.swift:139`） | `Text("菜单")` | 可见 UI 文案 |
| `CoreMenuButton`（原 `MenuButton.swift:169`） | `.accessibilityLabel("菜单")` | 可访问性标签 |
| `BookCover.swift:23` | `"未命名"` | 占位文案 |

本任务把这四处统一为英文，并把全库 UI 字符串纳入 **String Catalog（`.xcstrings`）**，使库具备可本地化能力（而非仅仅"改成英文"）。已是英文的那几处同样纳入目录，口径才算统一。

## Technical Details

### 关键前置：`Package.swift` 的 `defaultLocalization`（跨任务触碰）

**SPM 对含 `.xcstrings` 的 target 强制要求 `Package.swift` 声明 `defaultLocalization`**——不声明则包解析直接失败（`manifest property 'defaultLocalization' not set`）。

这是 owner 矩阵中一处**未建模的跨任务触碰**：`Package.swift` 归 **001**（`defaultIsolation`、`platforms` 枚举形式）。时序上无冲突（001 是阶段 0 硬前置，本任务在阶段 5），但两者改同一文件，故本任务 `depends_on` 含 001、`conflicts_with` 含 001。

约束：
- 本任务**只加 `defaultLocalization: "en"` 一行**，不碰 001 在该文件里落的 `swiftSettings` / `platforms` 改动
- 加完后须用**四条 trait 命令完整复验**（不是只跑 `swift build`）——`defaultLocalization` 影响资源打包路径，Blossom 构建同样要验

### 资源加载约定

`Localizable.xcstrings` 放在 `Sources/CoreDesign/Resources/` 下，随现有的 `.process("Resources")` 一并处理。**所有查找必须显式传 bundle**：

```swift
Text("menu.title", bundle: .module)
```

SwiftUI 默认走 main bundle，库内不传 `bundle: .module` 会在下游 app 里静默取不到译文（回落成 key）。这与 CLAUDE.md「所有资源查找都必须传入 `bundle: .module`」的既有约定一致。`.accessibilityLabel` 等接收 `LocalizedStringKey` 的 API 同样须走 `LocalizedStringResource(_:bundle:)` 或等价形态。

### `MenuButton` 已被 003 改名

**本任务须基于改名后的代码**：003 已把 `MenuButton` → `CoreMenuButton`（避开 macOS 上 SwiftUI 同名的 deprecated 类型）。审计清单里的 `MenuButton.swift:139,169` 是审计时的坐标，行号与类型名在 003、010 之后都会漂移，须按符号定位而非按行号。
