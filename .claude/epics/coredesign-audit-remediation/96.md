# Task: 按钮体系 + Sidebar 收敛

## Description

把四个 ButtonStyle 与 `Sidebar` 四种 row 中的结构性重复收敛掉，承载审计项 **B2b、B3a、B3b、B3c、B3d、B3e、B5、B8a** 共 8 项（对应 FR-4 的按钮 / Sidebar 部分，外加 FR-3 声明但归本任务实施的 B2b）。

本任务属于 epic 的**并行窗口 1**：owner 矩阵逐对验算确认 **005 ∩ 006 = ∅**（触及文件交集为空），因此可与 006（死代码清理）真正并发执行。这一段并行直接缩短 epic 最长的关键路径。

本任务同时是 004（Dynamic Type 改造）排在其后的收益来源：`Sidebar.swift` 现有 **16 处 `CoreTypography.` 引用**，是全库最高的单文件消费者；骨架收敛后应缩到**约 6 处**，004 的改动面随之显著下降。四个 ButtonStyle 的 font 行同理——B3d 把它们收敛成单一共享 modifier 后，004 的 `CoreControlMetrics.font(for:)` → `fontToken(for:)` 改造只需改一个调用点而非五个。

**B2b 的归属说明**：PRD FR-3 把「`Sidebar` 四处 `frame(height:)` → `minHeight`」写在 Dynamic Type 名下，但审计项 B2b 的实施**归本任务**，不归 004。理由是两者改**同一批行**（`Sidebar.swift:121,183,238,287`），由本任务在收敛出新骨架时一并带上，可彻底消除该处跨 Issue 文件冲突。004 只声明需求，不承担实施。

## Technical Details

### SC-8 的测量边界（须严格按此口径自查）

`SidebarNavigationRow` / `SidebarUtilityRow` / `SidebarDocumentRow` / `SidebarTagRow` 四个类型的 **`var body` 块之和**（`public var body: some View {` 那行到其配对 `}`，含首尾两行）：

- **含**共享骨架类型的 `body` 块
- **不含** `public init`、`private let` 成员声明、结构体声明/收尾行
- **不含** `#Preview` 与文档注释

> **口径修正（#96 执行期，2026-07-20）。** 本节原写「**含**共享骨架类型与薄封装 init」，与「≤60 行」这个数字**来自不同口径**，拼在一起不可达——按原口径，四个 `public init`（各约 11 行）+ 14 个 `private let` + 结构声明/收尾行合计已约 79 行，**在写任何 body 之前就已超 60**。
>
> 决定性证据：实测四个 row 的 **body 块合计恰好 118 行**，与 PRD 原话「约 120 行降至约 50 行」的前半吻合——证明 SC-8 的数字自始就是按 body-only 推导的。96.md 改写时扩大了边界却沿用旧数字。本节自身列出的 B5 证据坐标（`Sidebar.swift:104-130,157-188,215-243,263-292`）同样是 body-only 区间，自我印证。
>
> 现改回 body-only 口径，**硬上限 60 行不变**（对应 PRD 的「约 50 行」）。完成时须在 PR 描述中给出实测行数与所用脚本。

### 依赖 003 的原因

003 承载 `BorderlessButtonStyle` → `CoreBorderlessButtonStyle` 改名（A3a）与 `ButtonRoleStyleRole` 三个调色板属性补 `public`（A2c）、`MenuButton` → `CoreMenuButton` 改名（A3b）。本任务改写的正是这批文件的内部实现，须在改名与 `public` 修复落地后进行，否则 B3a 的 `resolvedColor` 会建立在即将被改的可见性之上。

### 与 004 的分工边界

- 本任务**不做** `CoreControlMetrics.font(for:)` → `fontToken(for:)` 的签名改造，那是 004 的范围
- 本任务**只**把散落的 font 行收敛到单一 modifier / 单一调用点，为 004 让路
- 若收敛过程中发现某处 font 调用必须同时改签名才能编译，停下并在任务中记录，不要越界改 `CoreControlMetrics` 的公开签名

### 与 006 并行的硬约束

005 ∩ 006 = ∅ 是矩阵验算结论，**执行时不得破坏**：本任务不得触碰 006 的自有文件（`EmptyState.swift`、`View+SizeReader.swift`、`KeyboardHandling.swift`、`BottomInputBar.swift`、`CommentCard.swift`、`RefPill.swift`、`SegmentedControl.swift`、`TimelineItem.swift`、`BookCover.swift`、`CoreGradient.swift`、`BorderModifier.swift`）。若确有必要，须先与 006 协调而非直接改。

### 代码风格

遵循仓库既有约定：显式 `self.`、中英双语注释与 `// MARK: -`、`#Preview` 与组件同文件、不为 iOS 26 API 加可用性回退。
