## Technical Details

### CLAUDE.md 由本任务统一改一次（关键协调点）

CLAUDE.md 被 **002**（分层色彩描述随 `FunctionalColor` 重定义更新）、**006**（删去 `.getSize` 的通用辅助描述）、**011**（D16b 的 `.focusedExternally` 描述）**三方触碰**。epic Implementation Strategy 与 PRD 冲突表都建议**由本任务统一改一次**，避免三次编辑同一文件产生冲突。

因此本任务须核对：002 与 006 应留下的 CLAUDE.md 改动是否已落地；若未落地，一并在此改完，并在任务记录中列出「代 002 / 006 改了什么」。

### `Banner.swift` 的移动 vs 编辑冲突（D9）

`Banner.swift` 被 **002**（legacy status token 迁移）、**004**（`CoreTypography` 消费）、**010**（B8b `makeBody` 抽公共实现、B8e `StatusLevel` 合并）触碰，而本任务要**整文件移目录**。**移动 vs 编辑是所有冲突类型里最烈的一种**——git 会把它呈现为「删除 + 新增」，其它任务在旧路径上的改动会静默丢失。

约束：**本任务的 D9 必须在其它任务改完 `Banner.swift` 之后执行**，这正是 `depends_on` 含 002/004/010 的原因。移动时用 `git mv` 保留历史。

### `BottomInputBar.swift:219` 的 `44` 不可直接替换（易错点）

`44` 是 HIG 最小可点击区域的经典魔法数。**`CoreControlMetrics.height(for: .extraLarge)` 是 48，不是 44**——直接替换会静默改变布局。正确处置是提为命名常量（如 `minimumHitTargetSide`），并在注释里写明它是 HIG 值而非 metrics 序列内的档位。

### `Sidebar.swift` 的额外验证要求（关键）

D18 要给 `Sidebar.swift` 补 `#Preview`，而 **004 在该文件落了 Dynamic Type 布局断言**。该断言被 `#if os(iOS)` 包住，**四条 SwiftPM 命令下完全不可见**。

因此本任务**必须额外跑第 5 条验证命令**：

```bash
xcodebuild test -scheme CoreDesign -destination 'platform=iOS Simulator,name=iPhone 17 Pro'
```

epic「每个 Issue 的完成定义」第 4 条明确点名本任务与 010 两者。若 001 的 CI 降级到本地 pre-push 级别，这条命令须在本地手工执行并记录输出。

### Preview 是本仓库的主要视觉冒烟手段

CLAUDE.md 明确：`#Preview` 块与对应组件放在**同一文件**中，它们不是测试，但是组件的主要视觉冒烟检查方式。补 Preview 时须与仓库既有形态一致（同文件、覆盖主要配置组合），`Sidebar` 的 6 个 public 组件应各有可辨识的预览场景而非单一堆叠。
