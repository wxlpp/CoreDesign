# Task: FocusRingModifier 文件骨架 + iOS 端实现

## Description

创建 `Sources/CoreDesign/Modifier/FocusRingModifier.swift`，含 iOS / iPadOS / visionOS 端的 `.overlay(RoundedRectangle().stroke())` 纯视觉实现。macOS 分支以 `#if canImport(AppKit)` 占位（与 iOS 相同的 overlay 兜底，**注释明确"待 Task 8 替换为 NSViewRepresentable"**），留给 Task 8 填充 NSFocusRing 集成。

## Acceptance Criteria

- [ ] 新建 `FocusRingModifier.swift`
- [ ] 暴露：

```swift
public extension View {
    func focusRing(
        visible: Bool = true,
        color: Color = .borderFocus,
        width: CGFloat = CoreBorderWidth.thick,
        cornerRadius: CGFloat = CoreRadius.medium
    ) -> some View
}
```

- [ ] iOS / iPadOS / visionOS 分支：`.overlay(RoundedRectangle(cornerRadius: cornerRadius).stroke(color, lineWidth: width))`，`visible == false` 时透明且不占布局
- [ ] macOS 分支以 `#if canImport(AppKit)` 提供占位实现：与 iOS 相同的 overlay 兜底，且文件顶部注释中显式说明"完整 NSFocusRing 系统集成由 Task 8 填充；当前为视觉近似回退"
- [ ] 提供 iOS `#Preview`，演示 focused / non-focused 两态（用 `@FocusState` 驱动）
- [ ] modifier 调用方在两端写法一致，`#if` 仅出现在 modifier 内部，调用方不需要写 `#if`

## Technical Details

- iOS 端键盘焦点由 SwiftUI `@FocusState` 自动驱动；本 modifier 接受外部传入的 `visible: Bool`，调用方按需绑定 `@FocusState` 或外部状态
- 占位实现的最小形式：

```swift
#if canImport(AppKit)
// TODO(Task 8): replace with NSViewRepresentable + NSFocusRing
@ViewBuilder
private func focusRingPlatformImpl(...) -> some View {
    self.overlay(RoundedRectangle(cornerRadius: cornerRadius).stroke(visible ? color : .clear, lineWidth: width))
}
#endif
```

- `visible == false` 时 stroke 用 `.clear` 或返回 `EmptyView`，避免任何视觉残留
