# Task: Toast（architecture spike）

## Description

新增 scene-scoped 全局通知组件 `Toast`。架构最复杂，是 epic 内唯一的 spike——`@MainActor @Observable final class ToastHost` + `EnvironmentValues` 注入 + `safeAreaInset` 渲染 + 队列状态机 + 手势 / 自动 / 点击 dismiss。

**不接受 singleton 妥协交付**（per epic ADR #9 + PRD R2 修订）：如 `@MainActor @Observable` + Environment 注入路径阻塞且 0.5d spike 内无解，本 task 应**作为 blocker 标记 + 升级给用户介入**，不降级到 singleton。

**本 task 内不再拆**（per epic ADR #14）：`ToastHost` / `ToastItem` / `ToastLevel` / `EnvironmentValues.toastHost` / `View.toastHost(edge:)` / 内部 `ToastView` 全部属同一 task 的实现细节。

## Technical Details

参考 PRD §FR-B-7 + epic.md `### 全局通知（Toast）`。

**Spike 内部节奏建议**：0.5d architecture spike（验证 `@MainActor @Observable` + EnvironmentValues + 单条 toast 渲染）+ 1.5–2.5d 实现（队列状态机 + dismiss 动画 + 手势 + 双平台 #Preview）。

**关键技术点**：
- `@MainActor @Observable` final class 在 Swift 6 strict concurrency 下：`@Observable` 不会自动赋 Sendable；本 task 不需要 `Sendable` 显式约束（`@MainActor` 已隐式提供线程安全）
- `Task.sleep` cancellation：内部维护一个 `dismissTask: Task<Void, Never>?`，每次显示新 toast 前 `dismissTask?.cancel()` 清理上一个
- `@Entry public var toastHost: ToastHost?` 默认值 `nil`：scene 没挂 `toastHost(edge:)` modifier 时 `@Environment(\.toastHost)` 拿到 `nil`，调 `nil?.show(...)` 是 no-op（建议 + 一行 `assertionFailure` debug warning）
