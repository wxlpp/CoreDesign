# Task: Tag

## Description

新增 GitHub 风格任意分类标签 `Tag`（chip / label，对应 GitHub issue labels）。

**职责边界**（与 Badge 区分）：Tag 是**任意分类标签**，颜色由调用方传入。**负例**：不得收敛成 5 个固定状态 level（不得有 `TagVariant` 枚举）；这是 Tag 与 Badge 的核心 API 差。

## Acceptance Criteria

### API 形态
- [ ] 新建 `Sources/CoreDesign/Components/Tag/Tag.swift`
- [ ] `public struct Tag<Label: View>: View`
- [ ] `public init(color: Color, removable: Bool = false, onRemove: (() -> Void)? = nil, @ViewBuilder label: () -> Label)`
- [ ] `Tag(_ text: String, color: Color, removable: Bool, onRemove: ...)` convenience init
- [ ] **不引入** `TagVariant` 枚举（per ADR / Badge 区分）

### 视觉
- [ ] 背景：调用方传入 `color` 的 ~12% opacity 衬底（参考 GitHub label 视觉）；具体 opacity 决策记入 doc-comment
- [ ] 文字色：调用方 `color`（直接用）或 `color` 的高对比变体；视觉抽查后决定
- [ ] 圆角：`CoreRadius.small`（值 3，per PRD FR-B-2，**不**用 `.full`，与 Badge 区分）
- [ ] 字号：`CoreTypography.bodySmallFont`
- [ ] padding：内边距走 `CoreSpacing.xs` / `sm`
- [ ] `removable: true` 时右侧加 `xmark.circle.fill` 系统图标 button，调用 `onRemove`；icon 尺寸走 `CoreControlMetrics.iconSize(for: .small)`

### 跨任务 guardrail

## Technical Details

参考 epic.md `### 简单结构组件`。Tag 视觉关键是"低饱和度衬底 + 高对比文字"——具体 opacity 公式（譬如 `color.opacity(0.12)` for bg + `color` for fg）需 visual-test 后确定。

`removable` 行为通过子 button 实现，**不**侵入 Label 视图本身——保持单一职责。
