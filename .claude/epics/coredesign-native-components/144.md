# Task: 视觉终审——实机复刻一屏 iOS 设置页

## Description

本 epic 全部交付物是**视觉组件**——机械判据(编译/测试/grep)证明不了它们好不好看。这是唯一能验证「与系统融为一体」的环节,是硬门不是可选(ADR-4)。

Phase 1 的视觉终审(#125)一次抓出四个只有实机截图才发现的缺陷:深色 Badge 语义色 α 太低不可见、预览宿主 320×480 兼容模式、`.preferredColorScheme` 泄漏、Tag 命中区把 chip 撑大 2.1 倍。本任务同等对待。

## 视觉终审结论（ios-visual-reviewer，2026-07-23）

**Verdict: PASS**（主交付设置页——达到"可发布的原生观感"，几乎无法与真机系统设置区分）。

逐项目视验收：
1. **分组容器浮于画布** ✅——深色纯黑下 InsetGroupedSection/Card 卡片是清晰的 `#1c1c1e` 抬升灰，与黑底明显拉开。**Issue #140 塌缩端到端修复验证通过。**
2. **图标方块** ✅——圆角连续、比例贴近 29pt 系统图标、白 glyph 居中、配色与 iOS 设置一致。
3. **分隔线 leading inset** ✅——有图标组从标题 leading 起（58pt），About 组对齐文本 leading，无错位。
4. **页眉/页脚** ✅——大写灰 footnote 分组头 + 次级灰页脚，符合惯例。
5. **`.tint` 覆盖** ✅——Notifications 开关绿、ProgressView 红填充、DisclosureGroup 红 chevron 全生效。
6. **AX5 最大字号** ✅——文字换行、行高撑高、不裁切不重叠；图标固定不放大是记录在案的取舍（保 inset 静态推导）。
7. **List 场景（AC9）** ✅——SettingsRow + `.listRowInsets(EdgeInsets())` 在原生 List 中无双重 inset、观感原生。

**处置的 finding**：ios-visual-reviewer 唯一 Improvement——`.core` demo 截图贴屏边/chevron 被裁，是截图宿主（launch-env 直达路径）缺水平边距、非组件缺陷。已在 `App/Sources/App.swift` 直达路径补 `.padding(.horizontal)`（设置页/List 除外，自带边距）并重截确认。无 Blocker，1 轮闭合（上限 2 轮内）。

**Nitpick（记录，不改）**：Card 走 `.surface(.content)` 带 1px 描边，而 InsetGroupedSection 无描边（iOS 原生做法）；属通用 card primitive 的合理取舍，若需像素级一致可后续提供无描边变体。
