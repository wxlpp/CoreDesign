# Task: 基础容器 Card / Separator / SectionHeader / SectionFooter

## Description

补齐 FR-11 的四个基础容器/分隔件。它们粒度小、成本低,是分组设置行(003)与视觉终审(005)的公共地基。

## Acceptance Criteria

- [ ] `Card`:`.surface(.content)` 的**具名封装**,提供默认内边距与标准圆角。不引入平行的容器体系——需要更细控制的场景仍直接用 `SurfaceModifier`。依赖 001 修好的 `.content` 背景,否则 Card 与画布同色
- [ ] `Separator(inset:)`:可控 inset 的分隔线。`inset` 至少支持「无 / leading 指定量」两种;默认 hairline 宽度,颜色走 `separator` 系统色
- [ ] `SectionHeader` / `SectionFooter`:分组的页眉/页脚。header 用 iOS 分组列表惯例(约 13pt、大写、`secondaryLabel` 灰);footer 用 `footnote` + `secondaryLabel`
- [ ] 全部 `public` 类型 / init / body 显式标注
- [ ] 各组件同文件 `#Preview`,深浅双模式
- [ ] `swift build` / `swift test` 通过

## Technical Details

`Card` 是薄封装,重点是别与 `SurfaceModifier` 形成平行体系——它就是 `.surface(.content)` + `.padding(默认值)`,不重新实现背景/描边/圆角逻辑。

`SectionHeader` 的大写与字号要对齐 iOS 原生分组列表(不是 demo app 的 `.sidebar` 风格)——Phase 1 视觉终审注意到 demo 用了 `.sidebar` list style 导致 header 非大写,这里要的是 `.insetGrouped` 惯例。

## Dependencies

依赖 001(`Card` 需要 `.content` 指向 raised)。与 004 无文件冲突(`Components/Card`·`Components/Separator`·`Components/Section` vs `Components/Style`),可并行。
