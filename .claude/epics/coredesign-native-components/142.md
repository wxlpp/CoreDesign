# Task: 分组设置行 InsetGroupedSection / SettingsRow

## Description

FR-10 的核心,也是本 epic 最能兑现「与系统融为一体」的一套——iOS 设置页 / 个人资料页 / 偏好面板的骨架。这是 Success Criteria #10 的主角:能不能不写任何 CoreDesign 之外的样式代码复刻一屏 iOS 设置页,取决于这一对组件。

## Acceptance Criteria

### InsetGroupedSection

- [ ] 圆角分组容器,视觉与 iOS `.insetGrouped` 一致(raised 背景浮在画布之上——依赖 #140)
- [ ] 可选页眉 / 页脚(复用 #141 的 `SectionHeader` / `SectionFooter`)
- [ ] **同一分组内相邻行之间的分隔线自动从图标右缘起始**(leading inset 对齐图标方块右缘),无需调用方计算。这是本任务最细的活
- [ ] 能嵌进已有的 `ScrollView` / `VStack`,不要求整页是 `List`(ADR-2)

### SettingsRow

- [ ] 图标方块(可着色,iOS 设置那种圆角色块 + 白图标)
- [ ] 标题 + 可选副标题
- [ ] 尾部 accessory 至少支持:value 文本、chevron、`Toggle`、**任意自定义视图**(用 `@ViewBuilder` 泛型 accessory)
- [ ] 行高在默认 Dynamic Type 下 ≥ 44pt,随字号增大而增高(不裁切)——用 `frame(minHeight:)` 地板
- [ ] 既能放进 `InsetGroupedSection`,也能直接作原生 `List` 的行(ADR-2)
- [ ] 尾部若挂 `Toggle`,该 `Toggle` 的强调色应能跟随 `.tint`(与 #143 的 `.core` toggleStyle 协同——不要在 SettingsRow 里写死 accent)

### 通用

- [ ] `public` API 表面完整;`#Preview` 深浅双模式,且至少一个 Preview 是多行分组(暴露分隔线 inset 对齐)
- [ ] 交互组件在 `.regular` 下命中高度 ≥ 44pt(测试覆盖,跑 iOS 腿)
- [ ] **圆角一律经 `CoreShape` / `ConcentricRectangle`,不出现裸 `RoundedRectangle(`**(PRD SC#3,Phase 1 已把 Sources 清零,新组件别重新引入)
- [ ] `swift build` / `swift test` 通过

## Technical Details

分隔线 leading inset 是最容易做错的:图标方块宽度 + 图标与标题间距 = 分隔线的 leading 偏移。这个值应从图标方块的实际尺寸推导,不是硬编码——否则图标尺寸一变就错位。参考 Phase 1 `Tag` 移除按钮那次的教训:布局相关的数值要么从 token 推导、要么实测,不凭字号推算。

`SettingsRow` 的 accessory 多态用 `@ViewBuilder` + 便利 init(像 Phase 1 的 `StateLabel` / `StatusRow` 那种 `where Label == Text` 便利 init 收 `String`)。
