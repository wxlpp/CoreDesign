# Task: Storybook UI（组件列表 + 详情页 + 组件数据）

## Description

实现 App 的 storybook UI：组件列表首页（`List`，按分类分组）+ 组件详情页（Light/Dark 双栏对比）。定义 `ComponentData` 枚举提供组件元数据与预览闭包。

## Acceptance Criteria
- [ ] App 首页为组件列表，按分类（Button、Form、Indicator、Layout 等）分组
- [ ] 点击组件进入详情页，展示组件名、描述、Light/Dark 双栏 preview
- [ ] Light/Dark 双栏通过 `HStack` + 左右各 `.preferredColorScheme(.light/.dark)` 实现
- [ ] 覆盖 13 个现有组件：Badge、Tag、Banner、BookCover、Button（4 styles）、BottomInputBar、MenuButton、Avatar、CheckBox、SegmentedControl、EmptyState、SearchField、ListRow、SidebarRow、UnderlinedTabBar、LabelIcon、ChevronRightIcon、DangerIcon、Toast
- [ ] 每个组件至少有一个带典型参数的 preview 实例
- [ ] Toast 组件在详情页提供一个 "Show Toast" 按钮来触发（Toast 需 scene-scoped host）

## Technical Details

### ComponentData 模型

```swift
enum ComponentCategory: String, CaseIterable {
    case button = "Button"
    case form = "Form"
    case indicator = "Indicator"
    case layout = "Layout"
    case navigation = "Navigation"
}

struct ComponentMeta: Identifiable {
    let id: String           // 组件英文名，如 "Badge"
    let name: String         // 中文显示名，如 "Badge 状态指示器"
    let description: String  // 一句话描述
    let category: ComponentCategory
    let preview: () -> AnyView
}
```

### 导航结构

```
NavigationSplitView (sidebar + detail)
├── sidebar: List(ComponentCategory.allCases) { category in
│     Section(category.rawValue) {
│       ForEach(components.filter { $0.category == category }) { comp in
│         NavigationLink(comp.name, value: comp)
│       }
│     }
│   }
└── detail: ComponentDetail(meta)
      ├── Text(meta.name + meta.description)
      ├── HStack {
      │     meta.preview().preferredColorScheme(.light)
      │     meta.preview().preferredColorScheme(.dark)
      │   }
      └── (Toast 特殊处理：Button "Show" → toast?.show(...))
```

### Toast 特殊处理
Toast 需要 `.toastHost(edge:)` 挂在详情页 root。提供一个 "Show Toast" 按钮触发示例 toast。
